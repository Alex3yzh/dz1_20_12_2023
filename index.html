• git clone - Клонує (завантажує) існуючий репозиторій з GitHub, з усіма файлами, гілками і комітами
  • git -h - справка
  • git config --global user.name - Встановлює ім’я користувача, що буде вказано для всіх ваших комітів
  • git config --global user.email - Встановлює поштову скриньку, що буде вказана для всіх ваших комітів
  • git add - Зберігає зміни в файлі, готуючись до створення коміту
  • git status - показує вам, в якій ви зараз гілці та інформує вас про те, що вона не розбіглася з такою ж гілкою на сервері
  • git commit - Записує зміни в файлі в історію версій, створюючи новий коміт
  • git log - Виводить історію версії поточної гілки
  • git push - Завантажує всі локальні зміни в гілках на GitHub
  • git branch - Створює нову гілку
  • git checkout - використовується для переключення гілок та отримання вмісту до вашої робочої директорії.
  • git pull - Оновлює локальну поточну гілку новими комітами з відповідної віддаленої гілки
  • git init - створює посилання з локального репозиторію на пустий репозиторій на GitHub
  • git branch -d - Видаляє вказану гілку
  • git merge - Поєднує історію вказаної гілки з поточною
  • git switch -c - Переходить на вказану гілку і оновлює робочу директорію
